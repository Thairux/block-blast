# Security Policy

## Reporting
Do not open public issues for security-sensitive reports.

Report security concerns privately to the repository owner with:
- affected area
- reproduction steps
- impact assessment
- suggested mitigation if available

## Scope
Security-sensitive areas include:
- InstantDB sync and profile data handling
- local persistence and exported diagnostics
- monetization hooks and purchase state
- third-party package and asset integrity

## Response Expectations
Initial triage should confirm receipt, severity, and next steps before public disclosure.
